Building a Neural Network from Scratch

Andrej Karpathy
calendar_today Oct 12, 2023

TLDR Summary

  • The video provides a comprehensive step-by-step guide on building a neural network completely from scratch using only Python and basic math libraries, avoiding high-level frameworks like PyTorch initially.
  • Karpathy emphasizes the importance of understanding backpropagation ("backprop") intuitively, demonstrating how gradients flow backwards through the computational graph to update weights.
  • A custom `Value` object is created to handle scalar values and their gradients, which serves as the fundamental building block for the `Micrograd` engine built in the tutorial.
  • The session concludes by training the scratch-built network on a simple classification problem, showing the loss decreasing over iterations and the decision boundary forming.

Introduction to Micrograd

[00:00:00] Hello everyone. Today we are going to build a neural network from scratch. Specifically, we are going to build a small autograd engine called Micrograd. It basically implements backpropagation. Backpropagation is this algorithm that really is at the core of how neural networks learn.

[00:01:15] What I think is interesting about Micrograd is that it's tiny. It's about 100 lines of code, maybe 150. But it supports everything you need to build and train neural networks. We're not going to use PyTorch or TensorFlow; we are going to build the machinery that is inside those libraries to understand how they work under the hood.

Understanding Derivatives

[00:05:30] Let's start with the definition of a derivative. If you have a function f(x), the derivative tells you how the function changes if you slightly bump the input. Formally, it's the limit as h approaches zero of (f(x+h) - f(x)) / h.

[00:08:45] We can verify this numerically. Let's define a simple quadratic function in Python and see what happens when we change x by a tiny amount, say 0.0001. You'll see that the slope we calculate matches the analytical derivative exactly. This intuition is crucial because neural networks are essentially just massive mathematical expressions that we differentiate.

Building the Value Object

[00:15:20] Now, let's create a class called Value. This class will wrap a single floating point number. But we want it to do more than just hold a number; we want it to keep track of its children—the other values that created it—and the operation that produced it (like addition or multiplication).

  • data: The actual scalar value.
  • grad: The gradient, which represents the derivative of the final output with respect to this value. Initialize it to zero.
  • _prev: A set of children nodes to build the graph.

[00:18:10] We will implement the `__add__` and `__mul__` magic methods so we can write expressions like a + b or a * b naturally in Python, but under the hood, we are building a computational graph.

Manual Backpropagation

[00:28:05] Before we automate it, let's do backpropagation manually. We have our expression graph. The output node, let's call it L (for Loss), has a gradient of 1.0 with respect to itself. Now we work backwards. If L = d + f, what is the derivative of L with respect to d? It's 1.0. The local derivative of addition acts as a router of gradients.

[00:32:00] For multiplication, say d = c * e, the derivative of d with respect to c is e. So the chain rule says we take the gradient coming into d from above, and multiply it by the local derivative. This is the essence of backprop: recursive application of the chain rule.
